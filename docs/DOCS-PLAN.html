1 卡片上你会看到什么

这一章是主入口。DOCS 加在现有项目卡片中间。下面每条都写清「要做什么」。

1.1

还没有 DOCS

要做：卡片显示「未检测到」；提供按钮「创建 DOCS」。

my-app Clean
DOCS
未检测到 Goal / Task 目录
手动提交 一键提交 Discard

点 创建 DOCS → 建好文件夹 + 空的 goal.md。

1.2

有目标，准备生成任务

要做：已有 goal.md、暂无 Task 时，显示主按钮「生成任务」。

my-app Clean
DOCS
暂无 Task · 已有 goal.md
手动提交 一键提交 Discard

生成任务 = 提示词模板 + goal.md → AI → 写出多条编号 Task 文件。

1.3

已有任务列表

要做：列出 Task（编号 + 标题 + 状态）；每条右侧有「⋯」。

my-app Clean

002 搭建页面骨架 待做 ⋯
003 输出报告 HTML 待做 ⋯
手动提交 一键提交 Discard

2 文件夹约定

项目根目录下的文件结构。

2.1

目录结构

要做：识别并使用以下约定路径。

你的项目/
└── DOCS/
    ├── Goal/
    │   └── goal.md ← 总目标
    └── Task/
        ├── 001-….md ← 一条任务一个文件
        ├── 002-….md
        └── 003-….md
2.2

没有 DOCS 时怎么办

要做：卡片提示缺失；一键创建空目录 + 空 goal.md（见 1.1）。

3 你怎么用（操作顺序）

把 §1 的状态串成一次完整使用。界面以 §1 为准，这里只记步骤。

3.1

写目标

要做：用户在 goal.md 写清项目目标（可先 1.1 创建，再编辑）。

3.2

生成任务

要做：用户点「生成任务」→ AI 拆成 001、002、003… 写入 Task 文件夹（界面见 1.2 → 1.3）。

3.3

打开某条任务

要做：⋯ → 打开；.md 见 1.5，.html 见 1.6。

3.4

实现某条任务

要做：⋯ → 实现 → AI 改代码 + 摘要写回 Task；过程中显示 1.7。

3.5

提交代码

要做：不自动提交；用户用卡片原有的「手动提交 / 一键提交」。

4 我们怎么做出来

开发实现顺序。可说「4.3 先别做」来砍范围。

4.1

认目录、读写文件

要做：扫描 DOCS、安全读写 Goal/Task 文件、支持创建空目录。

4.2

提示词模板（实现）

要做：在设置里落地 1.8：两套可编辑模板 + 恢复默认；保存进应用配置（与 aiProvider 一起）。

4.3

接 AI 通道

要做：生成任务 / 实现 都走现有 Codex 或 Cursor Agent 统一通道，不混用。

4.4

卡片 UI

要做：实现 §1 的全部卡片状态与更多菜单。

4.5

编辑器与打开 HTML

要做：接入 Vditor（对应 1.5）；HTML 用系统打开（对应 1.6）。

4.6

短文档

要做：补两份短说明：提示词约定 + 功能说明。

5 补充说明

默认不用记；有疑问再看。

5.1

「生成任务」默认提示词长什么样

要做：点「生成任务」时，实际发给 AI 的内容 = 模板 + goal.md + 项目现状。

设置里的模板 + goal.md + 项目现状 → AI → Task 文件

默认「生成任务」模板（可在 1.8 里改）

你是项目规划助手。请根据下方【项目目标】与【项目现状】，把目标拆成一组可执行任务。 你可以查阅公开网上资料作补充，但必须以目标与当前仓库情况为准，不要编造仓库里不存在的模块。 输出要求： 1. 只输出任务列表，不要开场白、不要总结 2. 每条任务足够小，一个人（或一次 AI 实现）能做完 3. 严格按下面格式重复多条： ### Task title: 不超过 20 字的标题 body: | - 要做什么 - 验收标准是什么 - 涉及哪些路径/模块（若已知） 【项目目标】 {{goal.md 全文}} 【项目现状】 项目路径：… 当前分支：… 工作区：干净 / 有未提交改动

「实现任务」另有一套模板（大意：按该 Task 正文在项目里改代码，做完用中文写摘要）。你在设置里也能改。

5.2

会不会自动提交

要做：会改代码、写回摘要；不会自动 Commit / Push（见 3.5）。

5.3

AI 用哪家

要做：设置里选 Codex CLI 或 Cursor Agent CLI；全部能力同一通道。
